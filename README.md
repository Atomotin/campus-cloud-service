# Campus Cloud Service

Простой учебный облачный сервис хранения файлов, сделанный на `Node.js` без внешних библиотек. Проект показывает базовую идею облачного хранилища: пользователь регистрируется, входит в свой аккаунт, загружает файлы через веб-интерфейс, а сервер сохраняет их и показывает только владельцу.

## Что умеет сервис

- загружать файлы через браузер;
- регистрировать пользователей и выполнять вход по логину и паролю;
- хранить сессию пользователя через cookie;
- хранить метаданные о файлах;
- показывать каждому пользователю только его файлы;
- скачивать файлы по HTTP;
- удалять файлы из хранилища;
- отдавать простой REST API.

## Структура проекта

- `server.js` — HTTP-сервер и API;
- `public/index.html` — интерфейс пользователя;
- `public/app.js` — логика фронтенда;
- `public/styles.css` — оформление;
- `storage/` — локальное хранилище файлов и индекса.
- `storage/users.json` — пользователи;
- `storage/sessions.json` — активные сессии.

## Запуск

1. Открой терминал в папке проекта.
2. Выполни команду:

```bash
npm start
```

3. Перейди в браузере по адресу:

```text
http://127.0.0.1:3000
```

## API

### Проверка состояния

```http
GET /api/health
```

### Регистрация

```http
POST /api/register
Content-Type: application/json

{
  "login": "student1",
  "password": "secret123"
}
```

### Вход

```http
POST /api/login
Content-Type: application/json

{
  "login": "student1",
  "password": "secret123"
}
```

### Выход

```http
POST /api/logout
```

### Получить текущего пользователя

```http
GET /api/me
```

### Получить список файлов

```http
GET /api/files
```

### Загрузить файл

```http
POST /api/files
Header: x-file-name=<имя_файла>
Body: бинарное содержимое файла
```

### Скачать файл

```http
GET /api/files/:id
```

### Удалить файл

```http
DELETE /api/files/:id
```

## Что можно сказать на защите

- Это облачный сервис хранения файлов с клиентской и серверной частью.
- Сервер реализован как REST API на `Node.js`.
- Пользователь сначала регистрируется или входит в аккаунт.
- После входа он видит только свои файлы, а не файлы других пользователей.
- Данные файлов сохраняются в `storage/files`, метаданные в `storage/index.json`.
- Пользователи и сессии сохраняются отдельно в `storage/users.json` и `storage/sessions.json`.
- Проект можно расширить базой данных, ролями и подключением внешнего облака.

## Идеи для улучшения

- добавить регистрацию и вход пользователей;
- хранить файлы в `PostgreSQL`, `MongoDB` или `S3`;
- сделать поиск по имени файла;
- ограничить доступ по ролям;
- добавить историю загрузок.
