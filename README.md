# Campus Cloud Service

Простой учебный облачный сервис хранения файлов, сделанный на `Node.js` без внешних библиотек. Проект показывает базовую идею облачного хранилища: пользователь загружает файл через веб-интерфейс, сервер сохраняет его, показывает список файлов и позволяет скачать или удалить файл.

## Что умеет сервис

- загружать файлы через браузер;
- хранить метаданные о файлах;
- показывать список загруженных файлов;
- скачивать файлы по HTTP;
- удалять файлы из хранилища;
- отдавать простой REST API.

## Структура проекта

- `server.js` — HTTP-сервер и API;
- `public/index.html` — интерфейс пользователя;
- `public/app.js` — логика фронтенда;
- `public/styles.css` — оформление;
- `storage/` — локальное хранилище файлов и индекса.

## Запуск

1. Открой терминал в папке проекта.
2. Выполни команду:

```bash
npm start
```

3. Перейди в браузере по адресу:

```text
http://127.0.0.1:3000
```

## API

### Проверка состояния

```http
GET /api/health
```

### Получить список файлов

```http
GET /api/files
```

### Загрузить файл

```http
POST /api/files
Header: x-file-name=<имя_файла>
Body: бинарное содержимое файла
```

### Скачать файл

```http
GET /api/files/:id
```

### Удалить файл

```http
DELETE /api/files/:id
```

## Что можно сказать на защите

- Это облачный сервис хранения файлов с клиентской и серверной частью.
- Сервер реализован как REST API на `Node.js`.
- Данные файлов сохраняются локально в папке `storage/files`.
- Метаданные сохраняются в `storage/index.json`.
- Пользователь может загрузить файл, увидеть его в списке, скачать и удалить.
- Проект можно расширить авторизацией, базой данных и подключением внешнего облака.

## Идеи для улучшения

- добавить регистрацию и вход пользователей;
- хранить файлы в `PostgreSQL`, `MongoDB` или `S3`;
- сделать поиск по имени файла;
- ограничить доступ по ролям;
- добавить историю загрузок.
